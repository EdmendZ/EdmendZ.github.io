CSEN 272 web search and information retrieval

Edmend Zhang2024-04-162024-04-20

Overview

search process

  1. 信息需求（Information Need）：这是指用户感知到需要某种信息的状态。这通常是搜索过程开始的起点。在图中，这用一个站在问号旁边的人来表示，表明有人有一个信息查询的需求。
  2. 表示（Representation）：在确定了信息需求后，用户需要以某种方式表达或者形式化这个需求。这通常涉及将需求转化为关键词或者查询语句。
  3. 查询（Query）：这是用户根据自己的信息需求形式化的搜索请求。它将作为输入提供给检索模型。
  4. 检索模型（Retrieval Model）：这是搜索过程中的核心组成部分。检索模型使用算法来解释查询并检索与查询最相关的结果。它作为连接用户查询和索引对象之间的桥梁。
  5. 索引对象（Indexed Objects）：这些是已经被搜索引擎处理过的信息对象，如网页、文章等。它们通过索引的方式被组织起来，以便检索模型能够高效地找到相关的对象。
  6. 检索对象（Retrieved Objects）：这些是检索模型根据用户的查询从索引对象中检索出来的结果。这个结果集合将被返回给用户。
  7. 评估/反馈（Evaluation/Feedback）：在用户接收到检索对象后，他们会评估这些结果是否满足他们的信息需求。这一步骤也可能包括用户对结果的反馈，这有助于搜索引擎优化其检索模型和索引。

Some core concepts of IR

  1. 查询表示（Query Representation）：
    • 桥接词汇差异（Bridge lexical gap）：这涉及处理不同词形或拼写变化的相同单词，例如，“system”和“systems”或“create”和“creating”。这通常是通过词干提取（stemmer）来实现的，将不同的单词形式转换为它们的基本形式。
    • 桥接语义差异（Bridge semantic gap）：这是指处理在意义上相近但表述不同的单词，例如，“car”和“automobile”。反馈机制（feedback）可以帮助系统理解这些术语在语义上的相似性。
  2. 文档表示（Document Representation）：
    • 文档内容的内部表示：这通常是指文档的内容如何在系统内部被表示和组织，例如通过倒排文档列表（inverted document list），这是一个索引，显示包含特定单词的所有文档。
    • 文档结构的表示：这是关于文档的不同部分如何被表示的，比如标题（title）、正文（body）等不同字段。
  3. 检索模型（Retrieval Model）：
    • 这里提到的是用于匹配用户查询和可用文档含义的算法。例如，向量空间模型（vector space model）和统计语言建模（statistical language modeling）是两种常用的检索模型。

Tentative topic

  1. 概述（Overview）：

    • 背景、历史、基本概念。

  2. 文本处理（Text processing）：

    • 齐普夫定律（Zipf’s Law）：描述词频分布的经验定律，认为在任何自然语言的语料库中，出现频率第二高的词，其出现频率大约是最高频词的1/2，第三高频的词频率大约是最高频的1/3，以此类推。
    • 分词（tokenizing）：将文本拆分成可操作的单元，通常是单词、短语或符号。
    • 词干提取（stemming）：将单词还原到基本形式。
    • 索引（indexing）：创建索引以便能够快速有效地检索文本数据。

  3. 查询操作（Query operations）：

    • 查询扩展（Query expansion）：增加额外的词或短语到搜索查询中，以改进检索结果的相关性。
    • 结构化查询（structural queries）：使用数据库查询语言，如SQL，来检索结构化数据。
    • 相关性反馈（relevance feedback）：根据用户反馈调整查询，改进未来的搜索结果。

  4. 经典检索模型（Classic Retrieval models）：

    • 布尔模型（Boolean model）：使用布尔逻辑（AND, OR, NOT）来匹配文本。
    • 向量空间模型（vector space model）：通过文档和查询向量的角度或余弦相似度来评价相关性。
    • TF-IDF加权（TF-IDF weighting）：一种在文档检索和信息检索中评估一个词在一个数据集或语料库中的重要性的数值统计方法。
    • 文本相似度度量（text-similarity metrics）：评估两段文本在意义上的相似度。

  5. 统计语言模型（Statistical language models）：

    • 包括用于处理文本信息的概率模型，例如PageRank和学习排名（learning to rank）算法。

  6. 推荐系统（Recommendation systems）：

    • 协作过滤（Collaborative filtering）：基于用户之间的相似性和历史偏好来推荐项目。
    • 基于内容的过滤（content-based filtering）：基于对项目内容的分析和用户的偏好来推荐项目。
    • 隐式用户反馈（implicit user feedback）：使用用户的行为数据来推测他们的偏好。

  7. 文本聚类和分类（Text clustering and categorization）：

    • 组织文本到有意义的、相似的群组。

  8. 高级主题（Advanced topics）：

    • 涵盖了信息检索领域更复杂或者最新的研究议题。
